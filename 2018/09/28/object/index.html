类的继承本质上是通过原型链实现继承。
借助构造函数实现继承

借助构造函数实现继承是通过call()或者apply()指定this来实现改变Parent1函数上下文来实现继承。但是不能继承Parent1函数原型上的方法。

借助原型链实现继承

这种方式是通过prototype来实现继承的，函数的原型指向new Parent2()。所以这种方法的缺点是继承的属性都是原型链上的，只要一个实例上继承的属性改变，就是改变原型链上的属性，导致所有实例属性都变了。

组合继承优化方式2
